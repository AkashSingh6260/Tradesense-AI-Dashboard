# Milestone 2: Market Data Collection and Preprocessing

## 1. Overview
The second phase of the TradeSense AI Pipeline establishes a robust data ingestion layer. Because machine learning models run exclusively on the reliability of the underlying metrics, this milestone strictly manages how financial OHLCV (Open, High, Low, Close, Volume) data is requested, cleaned, mathematically expanded, and distributed.

---

## 2. Collection of Historical Stock Market Data
* **Provider Selection:** Historical market data is sourced from **Yahoo Finance** through the Python `yfinance` REST layer. Yahoo Finance provides reliable, low-latency daily stock feeds without stringent rate limiting, making it excellent for autonomous AI scripts.
* **Volume & Breadth:** The engine downloads exactly **5 Years** of uninterrupted trading day metrics per stock symbol. For this prototype, prominent National Stock Exchange indices and tickers (e.g., `RELIANCE.NS`, `TCS.NS`, `HDFCBANK.NS`) are mapped programmatically to their backend endpoints.
* **Live Integration:** The script loops globally via `yf.download(ticker, period="5y")`, ensuring the models are trained dynamically on live datasets ending exactly on "Today".

---

## 3. Data Cleaning and Normalization
Raw financial data often contains imperfections resulting from public holidays, stock splits, or multi-index column structures. The pipeline enforces rigorous cleaning immediately upon collection:
1. **Flattening Multi-Indices:** Because internal parsers handle generic Pandas Series, the engine drops secondary hierarchical column levels generated by `yfinance` to prevent schema tracking errors.
2. **Handling Missing Values (`NaN`):** Due to standard closure days (weekends & holidays) or calculation overlaps, the engine invokes `df.dropna(inplace=True)`. Missing rows or rows corrupted by moving average initiation lag are actively jettisoned so the underlying `RandomForestClassifier` is trained strictly on complete multidimensional vectors.

---

## 4. Feature Extraction (Technical Indicators)
To empower the Artificial Intelligence with predictive leverage, raw Close prices are mathematically abstracted into deeper financial indicators using Pandas vectorization before compilation:

### 1. Trend Vectors (Moving Averages)
* **SMA 10:** A fast, 10-day Simple Moving Average tracking immediate market reactions.
* **SMA 50:** A slow, 50-day Simple Moving Average determining macro trend environments.

### 2. Momentum Metrics
* **RSI (Relative Strength Index):** Based on a 14-day rolling window of absolute gains and losses. It accurately tags momentum on a 0-100 gauge (values near 30 represent Oversold assets, values near 70 represent Overbought assets).
* **Daily Returns:** Percentage variations from yesterday's close (`df['Close'].pct_change()`). This serves as the underlying baseline for calculating algorithmic Volatility thresholds.

### 3. Creating the Training Target (The "Goal")
The engine computes tomorrow's movement today. It isolates logic predicting whether to output `Buy`, `Sell`, or `Hold`:
* It shifts the dataset backward (`df['Close'].shift(-1)`) to calculate tomorrow's Return.
* **Buy (+1):** Next Day Gain > 0.5%
* **Sell (-1):** Next Day Drop > -0.5%
* **Hold (0):** Otherwise sideways.

---

## 5. Storage and Output Handling
Once the data is cleaned and the algorithm is dynamically trained on the structured metrics, the insights are committed for continuous analysis:
1. **Model Cache:** During the script runtime, the `scikit-learn` algorithms utilize system RAM (memory) caching via Pandas DataFrames for split-second training iterations across massive 5-year OHLC blocks.
2. **Persistent API Delivery (SQLite):**
   * The AI output variables (`Recommendation`, `Trend`, `Confidence %`, `Risk`, and `Current Evaluated Price`) do not remain trapped inside the Python runtime.
   * They are committed directly via SQL execution (`UPDATE ai_recommendations SET recommendation = ?, trend = ?...`) into the local `database.sqlite` file.
   * This highly structured, decoupled SQLite table allows our Node.js backend to broadcast the AI judgments effortlessly to our React UI arrays multiple times per second.
